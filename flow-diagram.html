阶段 1 · 飞书消息接入
飞书事件入口（两种接入方式）
FeishuEventController POST /api/feishu/events receive(@RequestBody Map) 飞书 Webhook 回调入口：challenge 回显、消息解析、启动关键词欢迎卡、任务投递
FeishuEventController POST /api/feishu/card-events receiveCardEvent(@RequestBody Map) 飞书卡片回调：preset_question 发送预设问题、check_mcp_config 检查配置
FeishuWsEchoBot @PostConstruct start() · handleMessage(P2MessageReceiveV1) 飞书 WebSocket 长连接机器人（备选通道），与 HTTP 回调逻辑相同
消息解析
FeishuEventParser parse(Map body) → FeishuIncomingMessage 从飞书事件 JSON 提取 messageId / openId / chatId / chatType / text；群聊需 @机器人
FeishuClient replyWelcomeCard(messageId) 启动关键词触发欢迎卡（非启动关键词才投递任务）
任务投递
AgentTaskService createAndPublish(FeishuIncomingMessage) 写入 agent_task 表（PENDING）→ 发送 RabbitMQ 消息到 agent.task.query 队列；重复 messageId 忽略
↓
RabbitMQ EXCHANGE: agent.events · QUEUE: agent.task.query · DLQ: agent.task.query.dlq RabbitConfig: 2-8 并发消费者，失败进死信队列
阶段 2 · 任务消费 & 意图路由
RabbitMQ 消费
AgentWorker @RabbitListener(queues = "agent.task.query") handle(AgentTaskMessage) 消费消息：markRunning → 调用 AgentOrchestrator.process → markSucceeded / markFailed
核心编排入口
AgentOrchestrator process(String requestId) 加载 agent_task → 获取 question/messageId/chatId/openId/chatType → 意图路由 → 分发到各处理分支
意图分类
IntentRouter route(String question) → IntentRoute 关键词匹配 6 类意图： ① MCP_CONFIG_STATUS（配置状态，requiresMcp=false） ② SYSTEM_CONFIG（系统配置，requiresMcp=false） ③ WEATHER_EXTERNAL（天气外部，requiresMcp=false） ④ REPORT（报告，requiresMcp=true） ⑤ PROJECT_QUERY（项目查询，requiresMcp=true） ⑥ GENERAL_CHAT（兜底闲聊，requiresMcp=false）
阶段 3 · 多分支执行链路
IntentRouter 返回 IntentRoute 后，按以下条件分发
分支 A · FastGPT 引擎
if shouldUseFastGpt(route) == true
AnswerEngineRouter shouldUseFastGpt(route) → answerWithFastGpt(request) 当 AiRuntimeConfigService 配置为 FASTGPT 且意图非 SYSTEM_CONFIG / MCP_CONFIG_STATUS
↓
FastGptClient POST {fastGptBaseUrl}/v1/chat/completions answer(FastGptRequest, AiSettings) → FastGptResponse 调用外部 FastGPT，支持动态 baseUrl / 超时 / 记忆（chatId=群聊用chatId，单聊用openId）
↓
FeishuClient replyAnswerCard(messageId, question, answer, title, template) 回复飞书回答卡片 → 审计记录
分支 B · 非 MCP 路由
elif route.requiresMcp() == false
AgentOrchestrator processNonMcpRoute(...) 按意图分类处理： · SYSTEM_CONFIG → 配置提示文案 · WEATHER_EXTERNAL → DeepSeek 通用回答 + 天气未接入提示 · GENERAL_CHAT → DeepSeek 通用闲聊 · MCP_CONFIG_STATUS → 进入配置检查分支
↓
DeepSeekClient POST {deepseek.baseUrl}/chat/completions answerGeneral(question) → String 调用 DeepSeek 通用问答（model=deepseek-chat），不查项目数据
↓
FeishuClient replyAnswerCard(...) 回复飞书卡片 → 审计记录
分支 C · MCP 配置状态
elif category == MCP_CONFIG_STATUS
AgentOrchestrator processMcpConfigStatus(...) 检查 MCP 配置并实时验证每个项目 Token
↓
TokenResolver checkMcpConfig(openId, chatId, chatType, maxProjects) → resolveCandidates(...) 解析用户绑定的项目 Token 列表
↓
McpAdapter listTools(token) → MCP tools/list 对每个项目 Token 调用 MCP listTools 验证连通性
↓
FeishuClient replyAnswerCard(messageId, question, answer, "MCP 配置状态", template) 回复配置状态卡片 → 审计记录
阶段 4 · MCP 查询主链路（requiresMcp=true）
步骤 1 · Token 解析
TokenResolver resolveCandidates(openId, chatId, chatType, maxProjects) 按 owner（群优先 CHAT_ID → OPEN_ID → 旧版 PRIMELAYER_USER）查找绑定的 MCP Token；Token 加密存储，用 TokenCryptoService.decrypt 解密 项目匹配：精确 projectId → 项目名包含 → 拼音首字母（如 "lz" 匹配 "罗诊"）
TokenCryptoService decrypt(token) — AES-GCM-256 解密 主密钥存储在 system_config 表
步骤 2 · 工具发现 & 安全过滤
McpAdapter MCP JSON-RPC: POST {mcp.endpoint} listTools(token) → Map MCP 协议：initialize 握手 → notifications/initialized → tools/list；支持 SSE 响应解析
↓
McpToolRegistry filterDiscoveredTools(discoveredTools) → 白名单只读工具 只允许 get_/list_/query_/search_ 前缀；3 个内置兜底工具（primelayer.query_tasks 等）；拒绝 create_/update_/delete_ 变更工具
是否启用 agent-service？(app.agent-service.enabled)
路径 1 · Agent Service 编排（主路径）
if agentServiceClient.isEnabled() == true
AgentServiceClient POST {agentService.baseUrl}/v1/agent/answer answer(AgentAnswerRequest) → AgentAnswerResponse 多轮编排（最多 4 轮），每轮将上一轮 toolResults 回传给 agent-service
↓
AgentOrchestrator executeAgentToolCalls(...) → callAgentToolSafely(...) Java 后端执行 agent-service 返回的 ToolCall，调 MCP（含分页最多 50 页），结果回传
McpAdapter callToolWithPagination(token, toolName, args, pageSize) 支持 page/pageSize 与 offset/limit 两种分页风格
↓

① CLASSIFY classify_skill 关键词匹配路由到 5 个技能
project_report / project_status_qa / task_risk_qa / weekly_report / general_mcp_qa
→

含"全部"→全部项目
别名解析兜底（置信度 0.75-0.95）
→
③ PLAN plan_tool_calls 项目工作流(多轮)→
领域工作流(表单发现)→
单工具兜底
→
④ SUMMARIZE summarize_answer 领域管理报告→
DeepSeek 模型汇总→
确定性兜底汇总
→
⑤ CHECK quality_check 强制追加
"数据范围：基于本次 MCP 工具返回结果"
防止幻觉
agent-service 返回 toolCalls 时 → Java 执行 MCP 调用 → 结果回传 → agent-service 下一轮 → 返回 answer 时结束
↓
FeishuClient replyAnswerCard(messageId, question, answer, route.title(), route.cardTemplate()) 回复飞书回答卡片
路径 2 · DeepSeek 多工具规划（降级路径）
fallback（agent-service 未启用或调用失败）
DeepSeekClient POST {deepseek.baseUrl}/chat/completions planMultiTool(question, availableTools) → DeepSeekPlan LLM 规划 0-5 个工具调用（API key 未配置时走 heuristicPlan 启发式兜底）
↓
McpAdapter callToolWithPagination(token, toolName, args, pageSize) 对每个工具 × 每个项目 Token 执行分页调用（最多 50 页/工具）
↓
DeepSeekClient analyzePerTool(toolName, allPageResults) → PerToolAnalysis 阶段 1：逐工具全量数据分析
↓
DeepSeekClient analyzeCross(question, perToolAnalyses) → String 阶段 2：跨工具交叉综合分析，生成最终答案
↓
FeishuClient replyAnswerCard(messageId, question, finalAnswer, ...) 回复飞书回答卡片
阶段 5 · 审计日志 & 链路追踪（所有分支结束时并行写入）
主审计日志
AuditService writeMain(requestId, openId, chatId, primelayerUserId, projectIds, question, intent, answer, latencyMs, error) 写入 agent_audit_log 表（upsert），记录主审计信息
工具调用日志
AuditService writeTool(requestId, projectId, primelayerUserId, toolName, arguments, status, latencyMs, error) 写入 agent_tool_call_log 表，每次 MCP 工具调用记录一条
模型调用日志
AuditService writeModel(requestId, model, purpose, inputSummary, outputText, status, latencyMs, error) 写入 agent_model_call_log 表，记录 DeepSeek / FastGPT / agent-service 调用
链路追踪

后台关键交互链路
人员配置向导：PeopleConfigView → adminApi.saveUserBinding → AdminController → AdminService → AdminRepository
Token 验证：PeopleConfigView → adminApi.verifyProjectToken → AdminController → AdminService.verifyToken → McpAdapter.listTools（探活）
Token 保存：PeopleConfigView → adminApi.saveProjectToken → AdminController → AdminService → TokenCryptoService.encrypt → AdminRepository.upsertProjectToken
AI 引擎切换：SystemSettingsView → adminApi.saveAiSettings → AdminController → AiRuntimeConfigService.saveSettings → system_config 表
链路查看：AgentTasksView → viewTrace(requestId) → ChainTraceView → adminApi.getChainTrace → AdminController → ChainTraceService.load → Mermaid 流程图渲染
端到端测试：TestCenterView → adminApi.debugAgentQuery → DebugController → DebugService.queryAgent → AgentServiceClient / DeepSeekClient
MCP 工具测试：TestCenterView → adminApi.debugMcpTools → DebugController → DebugService.listMcpTools → McpAdapter.listTools
关键安全约束
MCP Token 不外泄：agent-service 只接收脱敏上下文和工具 schema，Token 加解密和 MCP 实际调用仅在 Java 后端执行
只读白名单：McpToolRegistry 只允许 get_/list_/query_/search_ 前缀工具，拒绝 create_/update_/delete_ 变更操作
数据范围声明：agent-service 的 quality_check 节点强制在答案末尾追加"数据范围：基于本次 MCP 工具返回结果"，防止幻觉
多轮上限：Agent Service 多轮编排最多 4 轮，超过则用 DeepSeek summarize 兜底
分页上限：MCP 分页调用最多 50 页，单次最多 5000 条数据
优雅降级：LangGraph 不可用时降级为函数链；agent-service 不可用时降级为 DeepSeek 多工具规划；DeepSeek API key 未配置时走启发式兜底
