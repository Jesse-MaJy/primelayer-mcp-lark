# 项目问答智能体增强 PRD

## 1. 文档信息

- 文档名称：项目问答智能体增强 PRD
- 当前版本：v1.0
- 日期：2026-07-02
- 适用系统：Lark Connect Agent Gateway / Python Agent Service
- 阶段：一期问答闭环增强
- 目标案例：用户询问“罗诊项目上个月的质量情况”时，系统能准确识别项目、时间、业务域、表单数据源，完成分页全量查询并输出管理报告式分析。

## 2. 背景

当前 Lark Connect 已具备飞书接入、Primelayer MCP token 管理、MCP 工具调用、Python `agent-service` 路由、Java 后端审计和链路追踪等基础能力。但在复杂项目问答中，现有系统仍偏固定流程：

- 相对时间理解不足，例如“上个月”未稳定转为绝对日期范围。
- 项目简称识别不足，例如“罗诊”需要映射到“罗氏诊断项目 / Roche”。
- 质量、安全、进度、风险等业务域缺少统一领域模型。
- 多表单选择仍依赖简单关键词，无法稳定区分核心表单和补充表单。
- MCP 列表数据需要全量分页，但 Python 规划层没有明确分页意图协议。
- 最终回答缺少稳定的管理报告结构、指标口径和效果评估体系。

因此需要增强系统 agent 的智能层，让其从“工具调用助手”升级为“项目数据分析助手”。

## 3. 产品目标

### 3.1 一期目标

建设项目问答智能体增强能力，实现：

- 支持质量、安全、进度、风险四类高频项目问答。
- 支持项目别名和简称识别。
- 支持常用相对时间解析，并在回答中明确绝对日期范围。
- 支持多表单自动发现、核心表单优先、补充表单合并分析。
- 支持分页全量查询意图，默认全量优先并受上限保护。
- 输出管理报告式答案：结论、关键指标、问题分布、趋势/风险、建议动作、数据范围。
- 建立离线评估样例集和指标体系，持续验证 agent 效果。

### 3.2 非目标

一期不做：

- 主动日报、周报、定时推送。
- 后台完整配置中心页面。
- 长期记忆和跨会话学习。
- 替代 Java 后端的权限、token、MCP 执行和审计边界。
- 让 Python `agent-service` 接触 MCP token。

## 4. 用户与场景

### 4.1 项目负责人

诉求：

- 快速了解项目质量、安全、进度、风险概况。
- 希望系统能自动识别项目简称和时间范围。
- 需要结论和建议，不只是一组原始记录。

典型问题：

- “罗诊项目上个月的质量情况怎么样？”
- “帮我看下本周安全隐患有哪些高风险项。”
- “当前项目进度风险是什么？”

### 4.2 一线执行人

诉求：

- 快速定位自己的问题项、待办、缺陷和整改状态。
- 需要看到关键明细和负责人信息。

典型问题：

- “我负责的质量缺陷还有哪些没闭环？”
- “本周我有哪些逾期整改项？”

### 4.3 管理员 / 运营人员

诉求：

- 通过样例集评估 agent 工具选择和回答质量。
- 维护轻量配置，例如项目别名、表单优先级、领域关键词。
- 通过审计和链路追踪排查错误。

## 5. 核心场景

### 5.1 罗诊项目上个月质量情况

用户输入：

```text
罗诊项目上个月的质量情况
```

系统应完成：

1. 识别“罗诊”为罗氏诊断项目 / Roche 对应项目。
2. 识别“上个月”为 2026-06-01 00:00:00 至 2026-06-30 23:59:59。
3. 识别业务域为质量。
4. 通过 `match_form_resource` 优先匹配：
   - 质量缺陷清单
   - 重点质量关注项
   - 其他质量相关补充表单
5. 对匹配表单调用 `query_form_data_list`，使用 pageSize=100 并启用自动分页。
6. 若某表单 total=353，则 Java MCP 执行层应拉取 4 页。
7. 必要时对关键记录调用 `batch_get_form_value_detail` 获取详情。
8. 输出管理报告式答案：
   - 总体结论
   - 缺陷/关注项数量
   - 严重程度或优先级分布
   - 整改状态分布
   - 高频问题或区域/责任人分布
   - 风险判断和建议动作
   - 数据范围和失败说明

### 5.2 本周安全隐患

用户输入：

```text
本周安全隐患有哪些高风险项？
```

系统应：

- 识别业务域为安全。
- 解析本周日期范围。
- 优先匹配安全隐患、安全检查、安全问题类表单。
- 输出高风险项、状态、责任人、整改建议和数据范围。

### 5.3 当前进度风险

用户输入：

```text
当前项目进度风险是什么？
```

系统应：

- 群聊中使用群绑定项目；私聊中根据项目线索或可访问项目判断范围。
- 识别业务域为进度和风险。
- 查询进度、计划、任务、风险相关数据源。
- 输出进度偏差、阻塞项、影响范围、建议动作。

### 5.4 多项目风险汇总

用户输入：

```text
帮我看一下所有项目的逾期风险
```

系统应：

- 识别范围为用户所有可访问项目。
- 只查询有有效 MCP token 的项目。
- 单项目失败不阻断其他项目。
- 最终回答说明成功项目、失败项目和数据覆盖范围。

## 6. 功能需求

### 6.1 领域识别

系统应识别以下业务域：

| 业务域 | 关键词示例 | 典型数据源 |
| --- | --- | --- |
| 质量 | 质量、缺陷、整改、验收、关注项 | 质量缺陷清单、重点质量关注项 |
| 安全 | 安全、隐患、检查、事故、危险源 | 安全隐患清单、安全检查表 |
| 进度 | 进度、施工、日报、计划、滞后 | 施工日报、进度计划、节点任务 |
| 风险 | 风险、逾期、阻塞、延期、待办 | 风险清单、任务待办、问题清单 |

系统可识别多个业务域，例如“进度风险”同时命中进度和风险。

### 6.2 项目别名识别

系统应支持项目别名轻配置：

- 项目 ID
- 标准项目名
- 英文名
- 中文简称
- 常见口语简称

示例：

| 项目 ID | 标准名称 | 别名 |
| --- | --- | --- |
| roche | 罗氏诊断项目 | 罗诊、罗氏诊断、Roche、罗氏 |

匹配规则：

- 用户问题中命中项目 ID、项目名或别名时，优先选择该项目。
- 群聊场景默认使用群绑定项目，用户显式提到其他项目时仍需受权限校验。
- 多个项目同时命中且无法消歧时，应追问用户。

### 6.3 时间解析

系统应支持：

- 今天
- 昨天
- 本周
- 上周
- 本月
- 上月 / 上个月
- 近 N 天
- 本季度
- 上季度

要求：

- 所有相对时间必须转为绝对起止时间。
- 默认时区使用 Asia/Shanghai。
- 回答中必须展示数据时间范围。
- 无明确时间时，按业务域使用默认时间：
  - 质量、安全、风险：默认近 30 天。
  - 进度：默认本周或当前周期。

### 6.4 表单发现与选择

系统应通过 `match_form_resource` 或 `list_form_resource` 发现候选表单。

选择策略：

1. 轻配置中的核心表单优先。
2. 与业务域关键词高度相关的表单优先。
3. 明显无关表单排除。
4. 核心表单和补充表单可合并分析。
5. 低置信度或候选冲突严重时追问用户。

质量域一期默认优先级：

1. 质量缺陷清单
2. 重点质量关注项
3. 质量检查 / 质量验收 / 质量巡检类表单

### 6.5 数据查询与分页

系统应默认全量优先：

- `query_form_data_list` 默认 pageSize=100。
- Python `agent-service` 在 `ToolCall` 中声明分页意图。
- Java 后端执行分页，Python 不直接调用 MCP、不接触 token。
- 默认最大页数 50。
- 默认最大记录数 5000。
- 超过上限时停止并在回答中说明截断。

分页失败处理：

- 单页失败不影响其他页、其他表单、其他项目。
- 最终回答必须说明成功页数、失败页数和可能影响。

### 6.6 详情增强

当列表数据不足以分析时，系统可调用 `batch_get_form_value_detail`：

- 默认只对关键记录取详情。
- 关键记录包括高严重度、未闭环、逾期、重复出现、模型认为需要展开的记录。
- 详情调用也必须受最大条数限制。

### 6.7 管理报告式回答

默认回答结构：

```text
结论：
关键指标：
问题分布：
趋势与风险：
建议动作：
数据范围：
未覆盖/失败说明：
```

质量域回答至少包含：

- 质量缺陷或关注项总量。
- 未闭环数量。
- 严重/高优先级数量。
- 高频问题类型。
- 责任人、区域或专业分布。
- 需要管理层关注的风险。

### 6.8 效果评估

一期必须建立离线评估体系：

- 不少于 30 条项目问答样例。
- 每条样例包含：
  - 用户问题
  - 期望业务域
  - 期望项目
  - 期望时间范围
  - 期望工具序列
  - 期望核心表单
  - 期望回答要点
  - 人工评分字段

指标目标：

| 指标 | 一期目标 |
| --- | --- |
| 工具选择准确率 | >= 90% |
| 时间范围解析准确率 | >= 95% |
| 核心表单选择准确率 | >= 85% |
| 回答包含数据范围 | 100% |
| 回答包含失败说明 | 100% |

## 7. 权限与安全

- Java 后端继续负责飞书身份、项目权限、MCP token 解密和 MCP 调用。
- Python `agent-service` 只接收脱敏上下文、项目列表、工具 schema、工具结果。
- Python 不接收、不存储、不日志打印 MCP token。
- 所有模型输入输出和工具调用记录必须经过敏感字段脱敏。
- 群聊必须使用提问者个人 token，不能使用群共享 token。

## 8. 验收标准

### 8.1 功能验收

- “罗诊项目上个月的质量情况”能完成项目、时间、业务域、表单、分页、汇总全链路。
- 安全、进度、风险各至少有 3 条样例通过。
- 多项目查询能说明成功和失败项目。
- 群聊查询能使用群绑定项目和个人 token。
- 表单候选低置信度时能追问。

### 8.2 质量验收

- Python 单元测试覆盖时间、项目别名、领域识别、表单排序。
- Java 单元测试覆盖分页执行和 DTO 兼容。
- 离线评估样例不少于 30 条。
- 所有最终回答包含数据范围。
- 部分失败时最终回答包含失败说明。

### 8.3 运维验收

- 链路追踪可查看每轮规划、工具调用、分页结果和最终汇总。
- 审计日志不包含 token 或敏感认证字段。
- 超过最大页数或最大记录数时有清晰截断标识。
