# 飞书回答长度上限调整设计

## 目标

将回答卡片正文的应用侧截断阈值从 3000 个 Java 字符调整为 10000 个字符。

## 范围

- 仅修改 `FeishuClient.safeText` 使用的长度阈值。
- 问题文本与回答文本继续共用同一套安全截断逻辑。
- 保留现有的截断提示“内容较长，已截断。”。
- 不调整飞书消息发送、卡片结构、分页或多消息拆分逻辑。

## 边界行为

- 长度小于或等于 10000：完整保留。
- 长度大于 10000：保留前 10000 个字符，并追加现有截断提示。

## 验证

增加卡片构建测试，覆盖 10000 字符不截断和 10001 字符截断两个边界，并运行飞书客户端相关测试及后端完整测试。
