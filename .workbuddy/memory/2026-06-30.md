## MCP 飞书侧调用失败 Bug 修复（BugFix 快捷路径）

### 问题
飞书侧问 "罗诊项目情况" 报错 "没有匹配该项目备注名的 ACTIVE MCP Token"，但管理后台测试中心手动选 Token 调 MCP 成功。

### 根因
1. **P0 直接原因**：agent-service 未运行时，AgentOrchestrator.processWithAgentService() 静默 fallback 到 processLegacy()
2. **P1 根本原因**：legacy 路径 TokenResolver.findTokenByHint() 只对 projectId/projectName 做字符串包含匹配；Token 表 projectName 是英文 "Roche"，用户口语 "罗诊" 匹配不上；数据库已有 project_remark 字段但代码没用

### 修复内容（3 文件）
- `backend/src/main/java/com/larkconnect/agent/token/TokenResolver.java`（主修改）
  - TokenEntry record 增加 projectRemark 字段
  - 3 处 SQL 查询加 project_remark 字段
  - findTokenBySingleHint 增强：6 种匹配（精确/包含/反向包含，覆盖 projectId/projectName/projectRemark）
  - findTokenByPinyinInitials 拼音首字母兜底匹配（GBK 区间映射，无第三方依赖）
  - buildNoMatchMessage 失败时列出可用项目列表
  - (d)/(e)/(f) 步骤加 !isEmpty() 守卫，修复空字符串边界 Bug（"anyString".contains("") 恒 true 问题）
- `backend/src/main/java/com/larkconnect/agent/admin/DebugService.java`（sanitizeTokenEntry 适配新字段）
- `backend/src/test/java/com/larkconnect/agent/token/TokenResolverTest.java`（H2 DDL 加 project_remark + 新增 9 个测试场景 A-I）
